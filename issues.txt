# 🧠 1. “No .env file found” → NOT an error

```bash
⚠️ No .env file found, using system environment variables
```

👉 This is actually **correct behavior in Kubernetes**

### Why?

* In local dev → you use `.env`
* In Kubernetes → you use:

  * `ConfigMap`
  * `Secrets`

And you already have:

```yaml

      name: post-config
```

👉 So your app **is using env vars**, just not from `.env`


👉 to:

```go
if os.Getenv("KUBERNETES_SERVICE_HOST") == "" {
```

✔ Only warn in local dev
✔ Silent in Kubernetes
