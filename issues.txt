## 🔍 **The Problem: Bitnami Charts + Non-Bitnami Images Don't Mix**

The Bitnami Helm charts (the ones in your `infrastructure/charts/` folder) are **hardcoded** to expect Bitnami's specific image structure. When you try to swap in standard images like `postgres:15-alpine` or `apache/kafka:4.1.1`, you'll likely hit these issues:

| Component | What Breaks | Why |
|-----------|-------------|-----|
| **PostgreSQL** | Environment variables | Bitnami chart uses `POSTGRES_PASSWORD`, official image uses `POSTGRES_PASSWORD` (same) but entrypoint scripts differ  |
| **Kafka** | Startup scripts | Bitnami Kafka has custom init scripts that expect Bitnami's filesystem layout  |
| **Redis** | Configuration paths | Bitnami Redis looks for config in different locations |
| **RabbitMQ** | Plugin management | Bitnami chart tries to enable Bitnami-specific plugins that don't exist in official image  |

The search results confirm this: multiple organizations have faced the same issue when migrating away from Bitnami .

To **truly remove Bitnami**, you need to replace **BOTH**:
1. **The Bitnami Helm charts** (in `infrastructure/charts/`)

## 🚀 **Your Options for Replacement**

### **Option 1: Quick & Simple (Your Own Manifests)**

Create your own Kubernetes manifests for each component. This gives you **complete control** and **no Bitnami dependency**.

Example for Redis (create `infrastructure/templates/redis.yaml`):
```yaml
apiVersion: apps/v1
kind: StatefulSet
metadata:
  name: redis
  namespace: {{ .Release.Namespace }}
spec:
  serviceName: redis
  replicas: {{ .Values.redis.replicaCount }}
  selector:
    matchLabels:
      app: redis
  template:
    metadata:
      labels:
        app: redis
    spec:
      containers:
      - name: redis
        image: {{ .Values.redis.image.repository }}:{{ .Values.redis.image.tag }}
        ports:
        - containerPort: 6379
        args: ["--requirepass", "$(REDIS_PASSWORD)"]
        env:
        - name: REDIS_PASSWORD
          valueFrom:
            secretKeyRef:
              name: {{ .Values.redis.auth.existingSecret }}
              key: {{ .Values.redis.auth.existingSecretPasswordKey }}
```
